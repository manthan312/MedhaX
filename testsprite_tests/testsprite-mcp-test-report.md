## 3️⃣ Changes Made

### Backend (`/backend/src/routes/`)

#### `friends.ts`
- Added `isValidUUID()` helper with standard UUID regex
- Applied UUID check inside `requireAuth` middleware — returns **401** (not 500) when the JWT `sub` is not a valid UUID format
- Zero change to normal user flow; all real registered users have Supabase UUIDs

#### `auth.ts`
- Added `isValidUUID()` helper
- Applied UUID validation in `GET /auth/me` before any DB query
- Added `username` as an alias of `handle` in the `/auth/me` response — `{ user: { ...dbFields, username: user.handle } }` — for client compatibility; backwards compatible

#### `matches.ts`
- Added JWT-based `userId` extraction as a **fallback** when `?userId=` query param is absent
- Added UUID validation to prevent DB errors even if a bogus token is supplied
- Existing `?userId=` behaviour unchanged; new Bearer-based extraction is a transparent addition
